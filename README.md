# TW-IP-Tool (Telegram Bot + Docker)

## 1) Установите Docker на сервер (Ubuntu 24)

```bash
sudo apt update
sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker
```

Проверка:

```bash
docker --version
docker compose version
```

## 2) Установите бота на сервер

### 2.1 Скачайте проект

```bash
sudo apt install -y git
git clone https://github.com/Daysema/TW-IP-Tool.git
cd TW-IP-Tool
```

### 2.2 Подготовьте данные
Создайте папку `data` (она примонтируется в контейнер как `/data`):

```bash
mkdir -p data/results
echo "[]" > data/accounts.json
```

В `data` со временем появятся (их не нужно создавать вручную):

- `accounts.json` — токены (можно править только через бота или осознанно вручную)
- `config.json` — настройки, в том числе **вкл/выкл авто-крутки** (`auto_spin`)
- `blacklist.json` — временная блокировка токенов API
- `stats.json` — **накопительная статистика** (создаётся после первых прогонов)
- `results/` — JSON-отчёты проверки аккаунтов (если используете сбор)

Токены удобнее добавлять в боте: меню **🛠 Токены**.

### 2.3 Создайте `.env` и заполните
Скопируйте пример и впишите свои данные:

```bash
cp .env.example .env
nano .env
```

Нужно заполнить:

- `TG_BOT_TOKEN` — токен бота от `@BotFather`
- `TG_ADMIN_USER_ID` — ваш Telegram user id (можно узнать у `@userinfobot`)

Опционально:

- `TG_CHAT_ID` — если хотите, чтобы служебные сообщения (логи warn/error, уведомления о балансе) всегда шли в конкретный чат

## 3) Запуск бота

```bash
docker compose up -d --build
docker compose logs -f
```

В Telegram напишите боту `/start`.

Остановить контейнеры:

```bash
docker compose down
```

## 4) Как обновить бота

```bash
cd TW-IP-Tool
git pull
docker compose up -d --build
```

## Как устроен бот сейчас

Откройте Telegram и отправьте `/start` — откроется панель с кнопками.

### Первая строка: авто-крутка и проверка аккаунтов

- **🔁 Авто-крутка** — единственный способ «крутить» поиск плавающих IP (отдельной кнопки «Поиск IP» нет).
  - **ВКЛ** — кнопка **зелёная**: бот сам запускает цикл создания IP, когда есть хотя бы один токен **вне** blacklist; если все токены временно в blacklist или список пуст — просто ждёт (или новых токенов, или истечения блокировок).
  - **ВЫКЛ** — кнопка **красная**: автозапуск выключен; если поиск уже шёл — он останавливается.
  - Состояние **сохраняется** в `data/config.json` (`"auto_spin": true/false`) и переживает перезапуск контейнера.
- **Проверка аккаунтов** — разовый проход по всем аккаунтам: лишние (нецелевые) floating IP удаляются, если включено в конфиге логики сбора.

Команда **`/stop`** останавливает активные задачи (поиск и проверку) и ставит **паузу автозапуска**: пока снова не включите авто-крутку (или не переключите её выкл/вкл), поиск сам не стартует.

### Вторая строка: Статус, Конфиг

- **Статус** — сводка и **накопительная статистика**. Счётчики по поиску и проверке **не растут вживую во время прогона**: они **добавляются к сумме только после завершения** соответствующего прогона и хранятся в `data/stats.json`, пока вы их не обнулите.
- Откройте **Статус** ещё раз — сообщение может обновляться каждые несколько секунд, пока вы на нём «сидите».
- **Конфиг** — целевые подсети (как в `config.json`).

### Третья строка: 🛠 Токены

- **➕ Добавить 1** — один Bearer-токен Timeweb строкой.
- **➕ Добавить пачкой** — `.txt` или вставка текста (по одному токену на строку).
- **🔎 Проверка на дубли**, **✅ Проверка правильности**, **🧹 Удалить токены**.

### Четвёртая строка: 🧹 Сброс статистики

Обнуляет **только** накопленные числа в `stats.json` (создано / удалено / найдено по поиску и по проверке). На токены, blacklist и `config.json` **не влияет**.

## Сообщения в чат

Итоги вроде «поиск завершён» и отдельные сводки по завершению **в чат не отправляются** — смотрите **Статус**. В чат по-прежнему могут приходить:

- пакеты **Log** с уровнями **warning / error** (если что-то пошло не так);
- **уведомления** (например, недостаточно средств на аккаунте по данным API).

## Чёрный список токенов

**Blacklist** означает, что токен на заданное время **не участвует** в поиске IP и **не используется** задачами бота — он пропускается, пока не истечёт срок (запись в `/data/blacklist.json`, переживает перезапуск контейнера).

- **Дневной лимит** API (`daily_limit_exceeded` и т. п.) — **24 часа**.
- **Нет средств / баланс / квота** — **1 час**.
- В прочих случаях блокировки по коду чаще всего **24 часа**, если не задано иное.

После истечения срока токен снова доступен для использования.
